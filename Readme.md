# Finder - File Search CLI Tool

Finderは、特定のディレクトリ内をスキャンし、指定した「拡張子」および「ファイルサイズ（KB単位）」に基づいてファイルを抽出するCLIツールです。

このプロジェクトは、Go言語による「関心の分離（Separation of Concerns）」と「クリーンアーキテクチャ」の設計パターンを学習・実践するためのサンプル実装です。

---

## 1. 機能

- 指定したディレクトリ以下の再帰的なファイル探索
- 拡張子によるフィルタリング（大文字小文字を区別しない）
- ファイルサイズによるフィルタリング（KB単位で指定可能）
- ファイル名、サイズ(KB)、最終更新日時のフォーマット出力

---

## 2. インストールと実行

### ビルド
$ go build -o finder main.go

### 実行例
$ ./finder -path ./samples -ext .jpg -size 100

### 引数（フラグ）
-path : 探索を開始するディレクトリパス（デフォルト: "."）
-ext  : フィルタリングする拡張子。ドットの有無は問いません（例: .png, jpg）
-size : 指定した数値(KB)以上のファイルのみを表示（デフォルト: 0）

---

## 3. 設計コンセプト：関心の分離

本アプリは、将来的な拡張性（例：クラウドストレージへの対応）やテストの容易性を確保するため、以下の3つの役割を分離して設計しています。

### Domain（ドメイン層）
- ファイル情報を表す純粋なデータ構造を定義。
- 外部ライブラリに依存せず、アプリの「共通言語」となります。

### UseCase（ユースケース層）
- 「拡張子が一致するか？」「サイズは閾値を超えているか？」といったビジネスルールを記述。
- 「どこからファイルを取得するか」はインターフェース（抽象）に任せ、具体的な詳細を知りません。

### Infrastructure（インフラ層）
- OSのファイルシステム（filepath.WalkDirなど）を操作する具体的な実装。
- ユースケース層が定義したインターフェースを満たすように実装されます。
